¿Qué usamos?

Un proyecto Java deliberadamente sencillo: sin Maven ni Gradle. Se compila con javac y las librerías viajan como .jar dentro de la carpeta lib/. Solo hace falta el JDK 17 para compilar, jugar y probar.

◆ Java 17

Es la base LTS. Aprovechamos varias comodidades modernas del lenguaje que hacen el código más corto y seguro:

  • records para datos inmutables como ResultadoAtaque o EfectoAplicado.
  • sealed interface en AccionTurno: la acción del turno solo puede ser Atacar, Defender o UsarObjeto.
  • switch con flechas y pattern matching de instanceof para ejecutar cada acción.
  • var y List.of(...) para escribir menos.

◆ Lombok

Genera en tiempo de compilación los getters y constructores repetitivos, para que el modelo se centre en las reglas y no en el "código de fontanería".

Sin Lombok

public final class Ataque {
    private final String nombre;
    private final int danioMin;
    private final int danioMax;

    public String getNombre(){ return nombre; }
    public int getDanioMin(){ return danioMin; }
    public int getDanioMax(){ return danioMax; }
    // ... un getter por cada campo
}

Con Lombok

@Getter
public final class Ataque {
    private final String nombre;
    private final int danioMin;
    private final int danioMax;

    // los getters los genera Lombok
}

También usamos @RequiredArgsConstructor para inyectar dependencias (consola, aleatoriedad, controladores) sin escribir el constructor a mano.

◆ JUnit 5

La lógica del juego está cubierta por 60 pruebas: daño dentro de rango, críticos, maná, efectos de estado, ítems, decisiones de la IA y balance de los personajes. La clave es que la aleatoriedad se puede fijar con una semilla, así los tests son reproducibles.

@Test
@DisplayName("un golpe crítico multiplica el daño por 1.5")
void ataqueConCritico() {
    Personaje atacante = nuevoPersonaje(100, 60, 1.0); // crítico garantizado
    Personaje objetivo = nuevoPersonaje(100, 60, 0.0);

    ResultadoAtaque r = atacante.atacar(BASICO, objetivo, new Aleatorio(1));

    assertTrue(r.critico());
    assertEquals(15, r.danio());      // round(10 * 1.5)
    assertEquals(85, objetivo.getVida());
}
Reproducible: new Aleatorio(1) siempre da la misma secuencia, así que un test que pasa hoy pasa siempre. Para eso Aleatorio tiene un constructor con semilla.

◆ GitHub Actions (CI)

Cada vez que se sube código, GitHub compila el proyecto y ejecuta toda la batería de tests automáticamente. Si algo se rompe, salta el aviso.

name: CI
on:
  push:
  pull_request:

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: "17"
      - name: Ejecutar la batería de tests
        run: ./test.sh
