# ¿Qué es?

Eliges un personaje y luchas en duelos uno contra uno. Cada personaje tiene 3 ataques, un especial poderoso y un estilo propio. El azar del daño, los críticos, las esquivas y los efectos hacen que cada combate sea distinto: gana quien mejor gestione su turno.

Modos

3 formas de jugar

Contra la CPU, dos jugadores en el mismo teclado o un torneo por rondas.

Táctica

Decisiones por turno

Atacar, defender o usar un objeto. El maná limita el especial.

Estilo

Terminal con arte ASCII

Paneles, barras de vida y maná y colores. Todo dentro de la consola.

# ¿Qué usamos?

Un proyecto Java sencillo, sin gestor de dependencias: se compila con javac y las librerías van como .jar en lib/.

Java 17lenguaje
Lombokmenos código repetido
JUnit 560 tests
GitHub ActionsCI en cada push
ANSIcolores en la terminal

Lombok

@Getter y @RequiredArgsConstructor eliminan getters y constructores repetitivos del modelo.

JUnit 5

Pruebas de daño, críticos, maná, efectos, ítems, IA y balance. La aleatoriedad usa semilla fija para ser reproducible.

GitHub Actions

Cada push compila y ejecuta toda la batería de tests automáticamente.

Sin build tool

Solo hace falta el JDK 17. Los scripts run.sh y test.sh compilan y lanzan todo.

# Cómo está montado

El código está separado por responsabilidades: el modelo no sabe nada de la consola, y la interfaz no sabe nada de las reglas. Así se puede cambiar el aspecto sin tocar la lógica y probar la lógica sin la interfaz.

src/com/tuio/juegoxturnos/
├── Main.java          # arranque
├── combate/           # flujo del combate
│   ├── Juego            # modos y menús
│   ├── Combate          # motor por turnos
│   ├── Combatiente      # personaje + control
│   ├── Controlador      # humano o CPU
│   └── AccionTurno      # atacar/defender/objeto
├── ia/                # EstrategiaCPU
├── modelo/            # reglas del juego
│   ├── Personaje, Ataque
│   ├── efectos/         # veneno, quemadura…
│   └── items/           # poción, escudo…
├── ui/                # Consola, Colores, ArteAscii
└── util/              # Aleatorio (con semilla)

Modelo

Personajes, ataques, efectos de estado e ítems. Objetos inmutables donde tiene sentido (Ataque, resultados).

Combate + IA

Un motor simétrico controla el turno de cada bando mediante un Controlador (humano o CPU). La CPU usa una estrategia sencilla.

Interfaz (ui)

Todo el dibujo —marcos, barras, arte ASCII y colores— vive aquí. El resto del juego solo le pide "muestra esto".

# Cómo jugar

Al arrancar eliges un modo y un personaje. Después, en tu turno, decides tu acción escribiendo un número y pulsando Enter.

Modos

1 jugador

Contra la CPU

Te enfrentas a un rival aleatorio controlado por la máquina.

2 jugadores

Local

Cada persona escribe su nombre y os turnáis en el mismo teclado.

Reto

Torneo

Vence en cadena al resto de personajes, recuperando algo de vida entre rondas.

En tu turno puedes…

  • Atacar — elige uno de tus 3 ataques. El daño varía un poco cada vez y puede ser crítico (×1.5).
  • Defender — reduces a la mitad el daño que recibas hasta tu siguiente turno.
  • Usar un objeto — del inventario, en lugar de atacar.

Efectos de estado

  • Veneno / Quemadura / Sangrado — daño al inicio de cada turno.
  • Aturdimiento — el enemigo pierde su siguiente turno.

Objetos

  • Poción de Vida (x2) — restaura 35 de vida.
  • Escudo (x1) — absorbe 30 puntos de daño.
  • Antídoto (x1) — elimina los efectos de estado.

Antes de cada acción se muestra a ambos combatientes con su arte, su vida y su maná, para ver de un vistazo cómo va el duelo.

# Cómo ejecutarlo

Necesitas JDK 17 o superior. Clona el repositorio y lanza el script:

# Compilar y jugar
./run.sh

# O a mano:
find src -name "*.java" > sources.txt
javac -cp lib/lombok.jar -d out @sources.txt
java -cp out com.tuio.juegoxturnos.Main

# Ejecutar los tests
./test.sh

El juego es interactivo y usa colores, así que córrelo en tu terminal (no en una tubería).
